## <a name="characteristics"> </a> Характеристика ділових процесів

*Дається опис бізнес-сценаріїв взаємодії бізнес-акторів, робітників і, можливо, інформаційної системи за допомогою наступної
специфікації:*

   
#### 1. ***ID ПРОЦЕСУ:*** USER__SIGN_UP
        
***НАЗВА:*** Створити обліковий запис
        
***УЧАСНИКИ:*** Система, користувач
    
***ПЕРЕДУМОВИ:*** Користувач перейшов на сторінку, але не має облікового запису
    
***РЕЗУЛЬТАТ:*** Створено акаунт для користувача
    
***ВИКЛЮЧНІ СИТУАЦІЇ:*** Акаунт вже існує, користувач вводить неправильні дані.
    
***ОСНОВНИЙ СЦЕНАРІЙ:***
1. Користувач робить запит на реєстрацію.
2. Система надає форму для заповнення.
3. Користувач вводить дані.
4. Система заносить користувача в базу даних.
5. Система створює акаунт користувача.
6. Система надає користувачу доступ до його акаунту.

---
   
#### 2. ***ID ПРОЦЕСУ:*** USER__SIGN_IN
        
***НАЗВА:*** Ввійти в обліковий запис
        
***УЧАСНИКИ:*** Система, користувач
    
***ПЕРЕДУМОВИ:*** Авторизований користувач перейшов на сторінку
    
***РЕЗУЛЬТАТ:*** Користувачу наданий доступ до акаунту 
    
***ВИКЛЮЧНІ СИТУАЦІЇ:*** Користувач вводить неправильні дані, відміна входу
    
***ОСНОВНИЙ СЦЕНАРІЙ:***
1. Користувач вводить дані.
2. Система обробляє дані.
3. Система надає користувачу доступ до акаунту.

---
   
#### 3. ***ID ПРОЦЕСУ:*** SURVEY__CREATE
        
***НАЗВА:*** Створити опитування
        
***УЧАСНИКИ:*** Замовник, система, користувач
    
***ПЕРЕДУМОВИ:*** Авторизований користувач знаходиться на сторінці свого акаунту
    
***РЕЗУЛЬТАТ:*** Створене опитування
    
***ВИКЛЮЧНІ СИТУАЦІЇ:*** Замовник відхилив створене опитування, користувач не створив жодного питання.
    
***ОСНОВНИЙ СЦЕНАРІЙ:***
1. Користувач переходить на сторінку, щоб створити опитування.
2. Користував заповнює форму опитування.
3. Система отримує та зберігає дані від користувача.
4. Cистема обробляє отримані дані і створює опитування.
5. Користувач завершує взаємодію із системою.

---
   
#### 4. ***ID ПРОЦЕСУ:*** SURVEY__UPDATE
        
***НАЗВА:*** Внести зміни до опитування
        
***УЧАСНИКИ:*** Замовник, система, користувач
    
***ПЕРЕДУМОВИ:*** Авторизований користувач знаходиться на сторінці свого акаунту
    
***РЕЗУЛЬТАТ:*** Змінене опитування
    
***ВИКЛЮЧНІ СИТУАЦІЇ:*** Замовник відхилив зміни, користував не зробив жодного питання 
    
***ОСНОВНИЙ СЦЕНАРІЙ:***
1. Користувач переходить на сторінку для внесення змін.
2. Користувач робить зміни.
3. Система вносить зміни до бази даних.
4. Користувач завершує взаємодію із системою. 

---
   
#### 5. ***ID ПРОЦЕСУ:*** SURVEY__DELETE
        
***НАЗВА:*** Видалити опитування
        
***УЧАСНИКИ:*** Користувач, система
    
***ПЕРЕДУМОВИ:*** Авторизований користувач знаходиться на сторінці свого акаунту
    
***РЕЗУЛЬТАТ:*** Видалене опитування
    
***ВИКЛЮЧНІ СИТУАЦІЇ:*** Відсутні
    
***ОСНОВНИЙ СЦЕНАРІЙ:***
1. Користувач переходить на сторінку опитування
2. Користувач робить запит на видалення опитування
3. Система видаляє опитування з бази даних
4. Користувач завершує взаємодію із системою

---
   
#### 6. ***ID ПРОЦЕСУ:*** USER__TAKE_SURVEY
        
***НАЗВА:*** Розпочати проходження опитування
        
***УЧАСНИКИ:*** Замовник, система, користувач
    
***ПЕРЕДУМОВИ:*** Користувач бажає пройти опитування
    
***РЕЗУЛЬТАТ:*** Результати опитування
    
***ВИКЛЮЧНІ СИТУАЦІЇ:*** неправильно введені дані, форма вже надіслана
    
***ОСНОВНИЙ СЦЕНАРІЙ:***
1. Користувач переходить на сторінку опитування
2. Користував вводить дані у форму
3. Система перевіряє дані від користувача
4. Система зберігає дані від користувача
5. Користувач завершує взаємодію із системою

---
   
#### 7. ***ID ПРОЦЕСУ:*** SURVEY__GET_RESULTS
        
***НАЗВА:*** Отримати результати опитування
        
***УЧАСНИКИ:*** Користувач, система
    
***ПЕРЕДУМОВИ:*** Авторизований користувач пройшов опитування та знаходиться на сторінці свого акаунту
    
***РЕЗУЛЬТАТ:*** Користувач отримує дані опитування
    
***ВИКЛЮЧНІ СИТУАЦІЇ:*** Користувач не пройшов опитування
    
***ОСНОВНИЙ СЦЕНАРІЙ:***
1. Користувач переходить на сторінку опитування
2. Користувач робить запит на отримання результатів опитування
3. Користувач переглядає результати
4. Користувач завершує роботу із системою


## Короткий огляд продукту

![Sample SVG Image](./logo.svg)

**Opinio** - надійний інструмент для проведення опитувань, який сприяє створенню, розсилці та аналізу опитувань. Користувачі можуть розсилати анкети респондентам по електронній пошті та розміщувати їх на своїх веб-сайтах і в соціальних мережах для підвищення відсотка отриманих відповідей. Подібно до цього, застосунок надає можливість проводити опитування щодо задоволеності клієнтів для збору відгуків про продукти та послуги, а також анкети щодо задоволеності працівників, щоб оцінити ставлення працівників до роботи в компанії. Інструмент дозволяє користувачам проводити опитування та перевіряти результати зі свого мобільного пристрою.
